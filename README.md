🚀 旅游之家（TravelFamilies）
📌 项目简介
旅游之家（TravelHome） 是一个专注于个人及家庭旅游场景的综合服务平台。旨在通过数字化手段，帮助用户发现心仪目的地、分享真实攻略，并提供完整的酒店预订与门票管理功能。

✨ 核心技术亮点
1. 强化的 Redis “黑白名单”安全机制
为了克服纯 JWT 无状态令牌无法实时注销的缺陷，在 Filter 拦截层实现了双重校验：

白名单（Token 维持）：Redis 实时存储活跃会话，支持多端登录挤下线及手动强制注销。

黑名单（实时风控）：针对异常账号实现一票否决，确保管理员封禁操作毫秒级生效。

2. 景点模块：高并发与数据一致性方案
多级缓存与击穿防护：针对 Top 10 热门列表开启 @Cacheable(sync = true)，通过本地锁机制防止瞬时击穿。

智能防刷机制：利用 Redis SETNX 实现 10 分钟访问冷却期，精准过滤 F5 恶意刷新。

Write-Back 异步落库：浏览量实时更新至 Redis，通过 @Scheduled 定时任务批量同步至 MySQL，降低数据库 IO 压力。

3. 工业级通用评论与多媒体组件
多维度解耦：采用 target_id 与 target_type 逻辑，一套代码支持景点、酒店、房型等多业务评价。

极致 SQL 优化：利用 Java 8 Stream 流在内存中实现数据分拣，确保无论评论多少，图片表的查询 IO 始终固定为 1 次（Anti-N+1）。

原子性计分：改用 Redis HINCRBY 原子递增，彻底解决多用户同时并发评分导致的数据覆盖问题。

4. 酒店预订与财务退款逻辑
动态库存搜索：支持用户按城市、入住/退房时间段进行空房筛选，确保预订不冲突。

阶梯退款算法：实现“提前退房”自动差价计算，根据实际入住天数与优惠券分摊比例，精确计算退款金额。

🛠 技术栈
核心框架：Spring Boot 3.x

安全鉴权：Spring Security + JWT

数据存储：MySQL 8 + Redis

持久层：MyBatis / MyBatis-Plus

云存储：阿里云 OSS（通用图片管理）

工具链：Lombok, Maven, Docker

📖 如何运行
环境准备：启动 Redis 与 MySQL 8 实例。

配置修改：在 application.yml 中配置数据库、Redis 地址及阿里云 OSS 密钥。

源码编译：执行 mvn clean install（注意：修改常量类后必须 clean 以刷新编译缓存）。

启动：运行 TravelFamiliesApplication。
